# 多项目投研工作区规则

本目录是多项目投研工作区，不是单一项目。进入子目录工作时，必须同时遵守全局规则、本文件，以及子项目自己的 `AGENTS.md`。

## 项目路由

- `xiaomi-ai-investment-system/`：小米 AI 硬件网络与期权价值研究。
- `kuaishou-kling-investment-system/`：快手可灵与快手 AI 视频投资判断系统。
- `ai-infra-investment-system/`：AI 基础设施版本 B 组合投研系统。
- `li-auto-investment-system/`：理想汽车投资方案设计与持续跟踪系统。
- `google-ai-investment-system/`：Alphabet / Google 股票投资方案和季度跟踪系统。
- `public/`、`server.js`：单股每日信息面板，当前以德翔海运 `02510.HK` 为样例，目标是可扩展成通用股票 HTML 面板。

开始任务前先判断当前工作属于哪个项目。不要把一个项目的数据、假设、估值或日志混入另一个项目，除非使用者明确要求做横向比较。

## 工作区共同规则

- 所有项目都继承全局数据生命线：来源、日期、口径、核实状态和过期状态必须清楚。
- 根目录只是工作区入口，不承载具体投资结论；具体结论写入对应子项目。
- 跨项目比较时，必须统一货币、时间窗口、估值口径和数据日期。
- 自动化、脚本和 HTML 面板只能生成研究输入，不得生成真实交易指令。
- 真实持仓、交易动作、对外发布、部署、提交和推送必须由使用者确认。

## 根目录单股面板规则

当修改 `public/` 或 `server.js` 时，目标是“单股每日信息面板”：

- 页面必须优先展示当前判断、关键价格、公告、新闻、研究观点和社区摘要。
- 每条关键证据必须保留来源和日期。
- 面板要适合每天打开复查，避免营销页风格。
- 修改后应启动本地服务并检查页面。

本地运行：

```bash
npm start
```

默认地址：

```text
http://localhost:5173/
```

如端口被占用，可使用：

```bash
PORT=5176 npm start
```
