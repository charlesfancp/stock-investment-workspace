# 方法论

本系统不是自动交易系统，而是自动投研判断系统。

系统负责收集数据、评分、触发预警、生成报告。最终买入、卖出、加仓、减仓和清仓必须由人工确认。

## 核心问题

系统每天或每周回答四个问题：

1. AI 基础设施主线是否还在上行。
2. 组合里的每只股票是否还符合原始买入逻辑。
3. 是否触发加仓、减仓、止损、止盈或假设破坏信号。
4. 组合是否需要从版本 B 调整到新版本。

## 评分结构

每只股票生成 0-100 分投资评分：

| 模块 | 权重 | 说明 |
| --- | ---: | --- |
| 产业景气分 | 25% | 所在 AI 主线是否继续上行 |
| 公司基本面分 | 25% | 收入、利润、订单、backlog、毛利率 |
| 估值赔率分 | 20% | 当前价格相对目标价、PE、PEG、涨幅、回撤 |
| 事件催化分 | 10% | 财报、产品发布、大客户订单、政策 |
| 风险扣分 | 10% | 监管、地缘、周期、客户集中、财务杠杆 |
| 技术位置分 | 10% | 趋势、均线、回撤、波动率 |

## 动作阈值

| 总分 | 动作 |
| ---: | --- |
| 85-100 | 强买入/加仓 |
| 70-84 | 买入/持有 |
| 55-69 | 持有/观察 |
| 40-54 | 减仓 |
| 0-39 | 清仓/回避 |

## 硬规则

如果原始投资假设被破坏，即使总分没有跌破阈值，也必须进入降级复核。

报告可以输出“允许加仓”“建议减仓”“触发止损复核”等判断，但不能自动下单。
