# Alphabet / Google AI 投资系统规则

本项目用于 Alphabet Inc. / Google 股票投资方案设计、季度跟踪、证据管理、评分和中文报告生成。系统定位是投研辅助和决策记录系统，不是自动交易系统。

## 与上层规则的关系

本项目继承全局 Codex 规则和 `New project/AGENTS.md` 多项目投研工作区规则。数据真实、来源和日期优先、结论可复盘、不得把无来源、无日期、未核实的数据写入结论区，这些底线不得被削弱。

## 项目对象

默认研究对象为 Alphabet Inc.，常见证券代码包括 GOOG / GOOGL。但以下信息必须先核实来源、发布日期或交易日、抓取日期、口径和核实状态，才能进入结论区：

- 可交易证券代码和交易市场；
- 最新价格、成交量、市值和股本口径；
- 最新财报、分部收入、经营利润、自由现金流和回购数据；
- 近期公告、业绩会、监管、反垄断、AI 产品、云业务和资本开支事件。

在未完成核实前，不得把任何股价、市值、股本、目标价、估值倍数、收入、利润、现金流、市场份额或 AI ROI 数据写入核心判断或投资结论。

## 第一版系统边界

第一轮目标：输入一行官方财报数据，输出一份有来源、有日期、有触发条件的 Google 投资判断报告。

## 阶段路线

后续开发必须按以下顺序推进，不得跳过数据纪律直接进入自动化：

1. Phase 2.5：决策日志去重，确保报告可反复生成但日志不重复堆积。
2. Phase 3：趋势图表，基于手工 CSV 生成可复盘图表。
3. Phase 4：HTML 决策面板，汇总结论、证据、估值、触发条件和决策日志。
4. Phase 5：手工录入真实估值快照后，第一次生成有 R/R 资格的投资判断。
5. Phase 6：再考虑自动抓 SEC / IR / 价格数据。

Phase 5 之前，不得因为经营数据强而生成真实买入、加仓、核心仓或仓位建议。

- 第一版只支持手工 CSV 输入财报和跟踪数据。
- 第一版不得联网抓取数据，不得自动调用行情、新闻、公告或第三方 API。
- 缺失字段统一显示“数据缺失”，不得猜测或补全。
- 抓取日期或发布日期超过 3 个月的数据必须显示“过期”。
- 只有已核实、未过期、带来源和日期的数据可以参与评分和结论。
- 2026Q1 数据必须先核实 Alphabet Q1 2026 earnings release、SEC Exhibit 99.1、Form 10-Q 等官方来源，再写入手工 CSV；未完成核实前只能作为待核实样例，不得进入高置信度结论。
- Cloud backlog 必须单独标注口径。若使用 2026Q1 backlog，必须注明 10-Q 披露的口径变化：报告开始纳入原始预期期限一年及以内的合同，因此不得把 backlog 环比大幅上升简单解释为纯订单加速。

## 固定技能顺序

本项目正式研究、季度更新、样例数据入库、报告升级和 HTML 面板更新，默认按以下顺序执行：

1. `source-integrity`：先核实官方来源、日期、报告期、口径、访问日期和结论资格。
2. `evidence-vault-rag`：把来源和事实写入证据库，建立 citation map 和 stale data check。
3. `variant-perception-thesis`：在证据基础上形成可验证、可推翻的投资命题。
4. Python 评分系统：读取手工 CSV，执行评分、信号和报告生成。
5. `decision-dashboard-system`：在报告和证据稳定后更新 HTML 决策面板。

不得跳过前两步直接把财报数字、股价、市值、backlog、FCF 或估值数据写入结论区。

## 评分框架

总分 100 分：

- Search 护城河：25 分；
- Cloud 第二曲线：35 分；
- AI ROI / FCF：25 分；
- 估值与仓位：15 分。

评分必须输出每个维度的得分、满分、数据质量状态和主要扣分原因。若关键输入缺失、待核实或过期，该维度应保守计分，并在报告中明确说明。

## 报告要求

自动生成的中文报告必须至少包含：

- 核心判断；
- 综合评分；
- 买入 / 持有 / 加仓 / 减仓 / 退出建议；
- Goldman、Damodaran、Dan Niles、Ackman/Berkshire 四个框架判断；
- 基准、乐观、风险三种情景和概率；
- 加仓、减仓、退出触发条件；
- 风险信号；
- 下季度跟踪清单；
- 数据来源、发布日期、报告期、抓取日期、口径和核实状态；
- 决策日志字段。

## 目录规则

- `evidence/`：证据库、来源索引、事实表、引用映射和过期检查。
- `data/raw/`：手工输入的原始 CSV，不直接视为已核实事实。
- `data/processed/`：脚本生成的中间结果。
- `research/`：投资命题、框架解释、专题研究。
- `reports/`：自动报告和人工复盘报告。
- `src/google_investment/`：评分、数据校验和报告生成代码。
- `scripts/`：命令行入口。
- `tests/`：pytest 测试。

## 安全边界

本项目不得自动下单，不得自动发送报告，不得自动发布网页，不得擅自修改真实持仓文件。涉及真实交易、对外发送、部署、提交或推送时，必须先说明影响并等待使用者确认。
