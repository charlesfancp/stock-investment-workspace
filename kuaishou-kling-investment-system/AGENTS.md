# AGENTS.md

## 与全局规则的关系

本项目继承全局 Codex 规则：来源、日期、过期标记、安全红线、决策日志和复盘要求不得被弱化。本文件只补充快手可灵项目的专属规则。

## 工作原则

本仓库是投资研究跟踪系统，不是自动交易系统。任何脚本、报告或模型输出都只能作为人工决策输入。

所有买入、加仓、持有、减仓、卖出输出，均为研究动作建议，不是自动交易指令。

## 数据规则

- 所有事实必须包含来源、发布日期和抓取时间。
- 未经核实的传闻不得进入 `config/valuation_assumptions.yaml`。
- 超过 3 个月的数据必须标注过期，除非它是长期假设或历史基准。
- 官方公告、财报、业绩会材料优先级高于媒体报道。
- 媒体报道必须标注可信度：高、中、低。
- 社区讨论只可作为情绪参考，不可直接进入估值模型。

## 报告规则

- 每日简报必须明确输出五类动作之一：买入、加仓、持有、减仓、卖出。
- 报告要区分“已验证事实”和“待人工确认信息”。
- 估值输出必须列明货币单位、关键假设和上行/下行空间。
- 投资主线发生变化时，必须更新 `decision_log/thesis_change_log.md`。

## 反证要求

每周必须主动寻找能推翻投资主线的证据，包括核心业务放缓、利润下修、AI 成本超预期、可灵融资不及预期、竞品冲击和监管风险。
